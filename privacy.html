Privacy Policy

Last updated: 2026-05-03

World Clock Climate ("the App") is a static web app that shows the local time and current weather for any point on a 3D globe. This document describes what information the App handles and how.

What we collect

The App does not run its own server, account system, or analytics. We do not collect, store, or transmit any personally identifying information.

  • Coordinates: The latitude/longitude under the on-screen crosshair are determined by the user's interaction with the globe in the browser. They are sent only to the third-party APIs listed below in order to look up a place name and weather, and they are not retained by us.
  • Browser language: Read locally to choose label language; never transmitted.
  • No cookies, no tracking, no advertising.

Third-party services

The App makes requests to the following services to provide its functionality. Each request includes the user's IP address and User-Agent as part of any normal HTTP request, governed by those services' own privacy policies:

  • OpenStreetMap Nominatim — reverse geocoding (coordinates to place name).
  • Open-Meteo — current weather conditions.
  • jsDelivr CDN — static library hosting (globe.gl, topology image).
  • GitHub (raw.githubusercontent.com) — country / admin-1 / city boundary data.

Local storage

The App may use browser cache and Service Worker cache to store static assets (HTML, JS, CSS, icons, map texture) for offline use and faster reloads. No user data is stored.

Children

The App is not directed at children, does not knowingly collect data from anyone, and contains no advertising.

Changes

If this policy changes, the "Last updated" date above will reflect the change. Changes are tracked in the public repository.

Contact

Questions or requests can be filed at the project's GitHub issues.
